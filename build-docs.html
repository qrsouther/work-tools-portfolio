Overview

The portfolio build system automatically scans all project directories in your workspace and regenerates the portfolio site with updated information from README files. This means you never have to manually edit HTML files to add or update projects!

How It Works

  1. Scans Projects: The build script looks through all directories in Documents/Code projects/
  2. Reads READMEs: For each project with a README.md file, it extracts:
    • Project title (from the first heading)
    • Description (from the first paragraph)
    • Technologies used (detected from keywords)
    • Status and stats (if present)
  3. Generates HTML: Creates:
    • Updated index.html with all project cards
    • Individual detail pages in projects/ folder
    • Proper badges based on detected technologies
  4. Preserves Styling: All existing CSS and design stays intact

Build Script Commands

npm run build

Rebuilds the entire portfolio site:

cd work-tools-portfolio
npm run build

What it does:

  • Scans parent directory for projects
  • Generates new index.html
  • Creates/updates project pages
  • Shows summary of changes

npm run rebuild

Rebuilds and prepares for Git commit:

npm run rebuild

What it does:

  • Runs the build script
  • Stages all changes with git add .
  • Shows git status so you can review changes

Step-by-Step: Adding a New Project

1. Create Your Project

Build your project in the parent directory (same level as other projects):

cd "Documents/Code projects"
mkdir my-new-project
cd my-new-project
# ... create your project ...

2. Write a README

Create a README.md file with at least:

# My New Project

A brief description of what this project does.

## Features

- Feature 1
- Feature 2

## Technologies

- Node.js
- React
- etc.

3. Rebuild the Portfolio

cd work-tools-portfolio
npm run build

4. Review and Commit

git add .
git commit -m "Add new project: My New Project"
git push

5. Wait for GitHub Pages

GitHub Pages will automatically rebuild your site in 1-2 minutes. Your new project will appear!

Troubleshooting

Project Not Appearing

  • ‚úì Ensure the project has a README.md file
  • ‚úì Check that the directory isn't in the exclude list
  • ‚úì Verify the README has at least a title (# heading)
  • ‚úì Run the build script and check console output

Wrong Information Displayed

  • Update the README.md in your project
  • Run npm run build again
  • Commit and push the changes

Build Script Fails

  • Ensure you're in the work-tools-portfolio directory
  • Check that Node.js is installed: node --version
  • Check console output for specific errors

Manual Override

If you need to manually edit a project page:

  1. Find the page in projects/project-name.html
  2. Edit the HTML directly
  3. Note: Your changes will be overwritten next time you run the build script

For persistent custom changes, modify the generateProjectPage() function in build-portfolio.js.

Files Created/Modified

Each build updates:

  • index.html - Main landing page with all project cards
  • projects/*.html - Individual project detail pages

Not modified:

  • styles.css - Your styling stays intact
  • build-portfolio.js - The build script itself
  • build-docs.html - This documentation page
  • README.md - Portfolio repository README
